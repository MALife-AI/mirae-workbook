# 워크북 사용 가이드 (신규 UI)

**미래에셋생명 · Claude Code 워크숍**
**대상: 학습자 + 강사 양쪽**

이 문서는 워크북 화면이 어떻게 구성되어 있고, 어떤 버튼이 무엇을 하며, 미션을 어떻게 풀고 채점받는지 설명합니다.
기존 `handout.md` 의 내용은 그대로 유효하고, 이 문서는 **새로 추가된 UI 기능**을 보충합니다.

## 1. 워크북 화면 한눈에

미션 슬라이드는 두 칸으로 나뉩니다.

```
┌─────────────────────────┬───────────────────────────────┐
│                         │                               │
│   왼쪽 (35%)            │   오른쪽 (65%)                │
│   안내 / 체크리스트      │   터미널 (claude 가 도는 곳)  │
│                         │                               │
│   - 🎯 GOAL             │   user01@workshop:~$          │
│   - 📥 INPUT            │   $ claude                    │
│   - 📤 OUTPUT           │   > PLAN.md 만들어줘          │
│   - 💬 프롬프트         │   ✓ Created PLAN.md           │
│   - 힌트 / 모범답안      │                               │
│   - ✅ 필수 체크         │                  ┌──────────┐ │
│   - ⭐ 도전 체크         │                  │ 🤖 AI 조수│ │
│   - 🤖 AI 채점 버튼     │                  └──────────┘ │
│                         │                               │
└─────────────────────────┴───────────────────────────────┘
                  진행 트래커 (1/8 ─ 2/8 ─ ... ─ 8/8)
```

오른쪽 하단에 항상 떠 있는 **🤖 AI 조수** 카드는 학습자가 작업하는 동안 자동으로 진행 상황을 살펴보고 짧게 코칭해줍니다.

## 2. 왼쪽 안내 패널 — 무엇을 입력하고 무엇이 나와야 하나

각 미션의 왼쪽 패널은 정해진 순서로 정보를 보여줍니다. 학습자는 위에서 아래로 읽으면서 따라가면 됩니다.

### 🎯 GOAL — 이 미션에서 배우는 것
미션의 핵심 목적. 한두 줄로 요약. 예) "Plan 모드를 써서 보고서 생성 워크플로우 4~5단계를 설계 + PLAN.md 로 저장"

### 📥 INPUT — 내가 무엇을 입력해야 하는가
학습자가 터미널에 어떤 명령/프롬프트를 넣어야 하는지 안내. 예) "터미널에서 claude 실행 후 /plan 슬래시 커맨드로 시작"

### 📤 OUTPUT — 어떤 산출물이 나와야 하는가
미션을 끝내면 어떤 파일이 생겨야 하는지 + 예상 형식.

- 파일 칩 (예: `📄 PLAN.md`) 이 함께 표시됨
- 파일이 **실제로 생성되면 칩이 회색→녹색**으로 바뀌고 글로우 효과
- 동시에 터미널 위에 큰 토스트로 "📄 PLAN.md 생성됨!" 표시 (4.5초)

### 💬 프롬프트
- **체험 슬라이드 (3.x)**: 미리 작성된 프롬프트가 노출됨. 복사하거나 ▶ 실행으로 바로 터미널에 넣을 수 있음
- **실습 슬라이드 (4.x)**: 직접 작성. 막히면 "모범답안 펼치기" 로 참고

### 힌트 / 모범답안
- **힌트**: 미션 진입 시 자동으로 펼쳐짐
- **모범답안**: 기본 접힘. 막혔을 때만 펼쳐서 참고

### ✅ 필수 체크리스트 (mandatory)
미션을 통과하기 위해 **꼭 만족해야 하는** 항목들. 예) "PLAN.md 파일이 프로젝트 루트에 생성되었다", "보고서 생성 단계가 3개 이상 있다"

- 파일이 생성되면 첫 번째 항목부터 자동 통과 표시
- AI 채점을 받으면 모든 항목이 ✓ / ✗ 로 평가됨

### ⭐ 도전 체크리스트 (challenge)
선택 항목. 더 좋은 결과물을 만들고 싶은 학습자를 위한 추가 기준. AI 채점 시에만 평가됨.

## 3. 버튼 사용법

| 버튼 | 기능 | 언제 쓰나 |
|---|---|---|
| **📋 복사** | 프롬프트를 클립보드로 복사 | 터미널에 직접 붙여넣고 싶을 때 |
| **▶ 실행** | 프롬프트를 터미널에 자동 입력 (Enter는 안 누름) | 빠르게 실행하고 싶을 때. 검토 후 직접 Enter |
| **🧹 클리어** | 터미널 화면 비우기 | 화면이 지저분해질 때 |
| **🤖 AI 채점 받기** | AI 가 산출물을 읽고 상세 평가 | 결과 파일이 만들어진 후, 더 자세한 피드백을 원할 때 |
| **🔄 다시 채점** | 이전 결과를 지우고 새로 채점 | 파일을 수정한 후 다시 평가받고 싶을 때 |

## 4. 미션 진행 흐름

학습자가 한 미션을 푸는 표준 흐름은 다음과 같습니다.

```
1. 슬라이드 진입 → 왼쪽 GOAL/INPUT/OUTPUT 읽기
       │
2. 프롬프트 복사 또는 ▶ 실행
       │
3. 터미널에서 claude 가 작업 → 파일 생성
       │
4. 자동 감지 (3초 폴링)
   ├─ 결과 파일 확인됨 → 📄 토스트 + 파일 칩 녹색 전환 + 컨페티
   └─ 자동 완료 → 다음 슬라이드로 (2초 후 자동 진행)
       │
5. (선택) 🤖 AI 채점 받기 → 필수/도전 항목별 ✓/✗ + 코멘트 + 점수
       │
6. (선택) AI 조수가 옆에서 짧은 격려/힌트 자동 표시
```

핵심: **체크리스트가 통과되면 자동으로 다음 슬라이드로 넘어갑니다**. AI 채점은 더 자세한 피드백을 원할 때만 누르면 됩니다.

## 5. 🤖 AI 채점 자세히

### 동작 방식
1. 학습자가 "AI 채점 받기" 클릭
2. 백엔드가 학습자의 산출 파일들을 읽음 (`outputFiles` 배열에 정의된 경로)
3. 미션의 **goal + inputDesc + outputDesc** 가 루브릭이 됨
4. 필수 + 도전 체크리스트가 평가 항목이 됨
5. **학습자 본인 크리덴셜**로 `claude -p --json-schema` 호출 (= 본인 쿼터 소모)
6. 60초 안에 응답 → 각 항목별 통과 여부 + 점수 + 요약

### 응답 형태
```
80점 ✓ PASS
"잘 작성된 PLAN.md 입니다. 단계가 명확합니다."

✅ 필수 체크리스트
  ✓ PLAN.md 파일이 프로젝트 루트에 생성되었다
    💭 확인됨
  ✓ 보고서 생성 단계가 3개 이상 있다
    💭 5단계가 명시되어 있어 충분합니다
  ✓ 각 단계가 무엇을 하는지 한국어로 설명되어 있다

⭐ 도전 체크리스트
  ✓ 입력 / 출력 / 산출물이 명시되어 있다
    💭 각 단계에 명확히 표시됨
  ⨯ 마크다운 헤딩이 사용되었다
    💭 번호 목록만 있어요. ## 헤딩을 추가하면 더 좋습니다
```

### 보안
- 학습자가 다른 사람 파일에 접근 못 함 (`sudo -u userXX` 로 본인 권한)
- 채점 중에는 모든 도구 차단 (`--disallowedTools "Bash Write Edit ..."`)
- bare mode (CLAUDE.md / hooks 자동 로딩 안 함)
- 60초 타임아웃, 8KB 출력 제한
- 입력 검증: rubric ≤ 4000자, files ≤ 8개, checklist ≤ 12개

## 6. 🤖 AI 조수 (어시스턴트 오버레이)

미션 슬라이드 우측 하단에 자동으로 떠 있는 작은 카드. 학습자가 작업하는 동안 옆에서 진행 상황을 봐주는 조수 역할.

### 동작
- 미션 진입 8초 후 첫 호출 (학습자가 화면 읽을 시간)
- 그 후 30초마다 자동 호출
- "지금 확인" 버튼으로 즉시 호출도 가능

### 무엇을 보고 코칭하나
- **터미널 스크롤백 (최근 1000줄)** — 학습자가 어떤 명령을 쳤고 결과는 뭐였는지
- **산출 파일 내용** — outputFiles 가 있으면 그 파일의 현재 내용
- **미션 컨텍스트** — goal + 필수 체크리스트

### 응답 종류
- 👍 **good** — 잘 진행 중
- ✨ **almost** — 거의 다 됨
- 🤔 **stuck** — 막힌 듯
- 💡 **empty** — 아직 시작 안 함

### UX
- 한 문장 격려/힌트 + (있으면) 다음 단계 한 줄
- 학습자가 닫거나 최소화 가능 (작은 동그라미 칩으로 축소)
- 실시간 코칭이라 응답 토큰 1024 / 30초 타임아웃 (가벼움)

## 7. 강사용 가이드

### 시연 슬라이드 자동 Haiku 전환
- "시연:" 으로 시작하는 슬라이드 진입 시 모든 사용자가 자동으로 **Claude Haiku 4.5** 로 전환 (응답 토큰 1024)
- 그 외 슬라이드에서는 **Claude Sonnet 4.6** (응답 토큰 8192)
- 시연 중 Claude 가 너무 길게 답해서 지루해지는 문제 해결
- 강사(user00)는 추가로 터미널이 자동 클리어되어 깨끗한 데모

### 학습자 미션 채점은 학습자 쿼터 사용
- AI 채점 / AI 조수 모두 학습자 본인 크리덴셜로 호출
- 강사 쿼터에 영향 없음
- 학습자가 채점 폭주 시 60초 타임아웃 + 1024 토큰으로 보호

### 대시보드 모니터링
- `admin` 계정으로 접속하면 모든 학습자 진행률 + AI 채점 사용량 확인 가능
- AI 채점 실패 시 학습자 화면에 빨간 박스로 표시 (서버 로그도 함께 확인)

## 8. 트러블슈팅

### "AI 채점 받기" 버튼이 회색으로 비활성화돼 있어요
→ 결과 파일이 아직 생성되지 않았기 때문. 먼저 프롬프트를 실행해서 파일을 만드세요.

### 채점 결과가 "채점 실패" 로 나와요
→ 보통 두 가지 원인:
1. 학습자 크리덴셜 만료 → `claude /login` 다시 실행
2. 파일이 너무 큼 (64KB 초과) → 파일을 줄이거나 일부만 평가

### AI 조수가 계속 "stuck" 으로만 떠요
→ 학습자가 터미널에 거의 입력을 안 하고 있을 때. 정상 동작.
→ 닫고 싶으면 우상단 ✕ 또는 − (최소화)

### 파일 생성 토스트가 안 떠요
→ autoChecks 가 false→true 로 바뀌는 순간만 표시.
이미 통과한 미션을 재방문하면 안 뜸 (의도된 동작).

### 시연 Haiku 가 적용 안 돼요
→ 슬라이드 제목이 "시연:" 로 시작해야 함. 다른 접두어는 무시.
→ 강사 본인 터미널은 자동 클리어되지만, 학습자는 다음 미션 슬라이드 진입 시 자동으로 normal 모드로 복귀
